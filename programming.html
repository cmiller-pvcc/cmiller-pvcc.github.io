Chapter 5: Programming Algorithms & Software Testing

Phases of computer program development:

      Phase 1: Understand the Problem
      Phase 2: Design an algorithm to solve the problem
      Phase 3: Write the program code in appropiate language based on the algorithm
      Phase 4: Enter the program and save
      Phase 5: Execute the program
      Phase 6:Test the program, find the errors and debug, then retest
      Phase 7: Document and evaluate the program

Programming structures:

      Sequential structure:Top-down execution
      Decision structures: if/then/else
      Iterative (looping):
      • While loop: continue looping as long as something is true
      • repeat/until loop: continue looping until something is true
      • for loop: loop a given number of times
      Recursive structures:
          Repeat a set of instructions as a subtask of itself
          rrepetition continues until:
          • degenerative case. which is a termination condition, is met
          • (degenerative case is also called a "base case")

Two clasic programming/database problems

      Sorting: Descending order/Ascending order
      • Sorting numbers
      • sorting strings (in accordance with ASCII values)
      Searching: find specific data that meet a certain criteria
          In database:
          • Filters: temporary search
          • Queries: search criteria are saved so they don't have to be recreated
      Two popular search techniques:
          Sequential search: search from top to bottom or bottom to top, one item at a time
          Binary search:
          • only works if data is SORTED first
          • start in middle, then go to half of that, etc.

Two classic program goals:

      Efficiency: Program takes minimal time and/or resources
      Correctness: Program runs in expected ways and works for all expected conditions

Software testing and verification

      Kinds of tests:
      • Acceptance testing: Verifies if the whole system works like it's supposed to
      • Integration testing: Makes sure that that software components or functions are both working together
      • Unit testing: Making sure that every software unit does what it's supposed to do
      • Functional testing: Evaluates the functionality of an application or system by testing its individual features or functions
      • Performance testing: Tests how the software operates from different workloads
      • Stress testing: Tests how much it can take before it fails.
      • Usability testing: Tests how well customers can use the system
      Software testing is done to detect:
      • Architectual flaws
      • poor design decisions
      • Invalid or incorrect functionality
      • Security vulnerability
      • Scalabilitiy issues
      Testing best practices:
      • Continous testing (does testing early and often. This happens throughout the whole code lifecycle)
      • configuration management (organizations maintain test assets and track what software builds to test)
      • Servic virtualization (simulates the missing services or systems that are not completed. allows the team to test sooner)
      • Defect or "bug tracking" (measures and improves quality)
      • Metrics and reporting (allows team members to share status, goals and test results)
