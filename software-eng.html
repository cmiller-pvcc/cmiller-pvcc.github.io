Software Life Cycle: development phase actual creation of the software
Requirement analysis (SLC) where the software requirements are collected, analyzed, and documented
Design (SLC) creating a plan for how the software will be developed
Implementation (SLC) the phase where the code is written
Testing (SLC) where the code gets tested
modular Implementation breaks down a system or smaller into a smaller componets
Unified Modeling Language (UML) the standard language for software systems
use case illistrtes the interaction with the user and the software.
use case diagram visual to show the interaction between users and software
Data relationships describe the way data points are interconnected or related to one another
one-to-one (DR) each record in one table corresponds to exactly one record in the other table
one-to-many (DR) a single record in one table is related to multiple records in another table
many-to-many (DR) multiple record in one table is related to multiple records in another table
design pattern simple solutions used for the common problems in software
software quality assurance (SQA) ensures that the software product meets or surpasses the standards
software testing evaluates the software syatem to make sure that it meets the requirments
glass box testing (ST) examines the internal structure of the software
black box testing (ST) examines th ebehavior of the software
beta testing (ST) sending out a limited edition version of the software to a small group to get feedback
alpha testing (ST) used by a limited group of users in a controlled enviroment before the software gets released to the public
Documentation written or visual representation to show how to use or understand the software
user documentation (D) written instructions or guides that give out information on how to use the software
system documentation (D) documents or written texts that give information about the systems design, functionality, development, and use
technical documentation (D) include user manuals, documentation, guides, and other technical materials that provide in depth information about the software
intellectual property law legal rules that protects the creation of inventions. allows them to control their creations
software license (IPL) legal agreement between the owner, software, and user that makes the user agree to the "terms and conditions"
software disclaimer (IPL) a statement or notice that is included in the software documentation that outlines the liabilities for the software provider.
